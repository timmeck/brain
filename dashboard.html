🔄 How It Works
1

Code Import & Analysis

Brain scans your projects and analyzes every source file. It extracts structural fingerprints, detects exports/imports, identifies the language, and scores each module for reusability across 6 signals.

2

Error Detection & Matching

When an error occurs (via hooks or manual report), Brain parses it with language-specific parsers (Node, Python, Rust, Go, Shell, Compiler, Generic), creates a fingerprint, and matches against known errors using 6 weighted similarity signals.

3

Synapse Formation

Every connection (error-to-solution, code-to-code, project-to-module) creates a synapse. Connections strengthen through Hebbian learning ("neurons that fire together wire together") and weaken through temporal decay.

4

Learning & Confidence

The learning engine tracks solution success rates using Wilson score intervals, clusters similar error patterns via centroid analysis, and automatically generates prevention rules when patterns stabilize.

5

Research & Insights

The research engine continuously discovers trends in your errors, identifies knowledge gaps, finds cross-project synergies (shared patterns), extracts template candidates, and generates actionable project suggestions.

6

Proactive Prevention

Brain surfaces insights through MCP tools and notifications. Before you write code, it checks for similar existing solutions. Before you hit an error, it warns about known patterns. It gets smarter with every interaction.

〉 CLI Commands
brain start
Start the Brain daemon. Initializes DB, runs migrations, opens IPC named pipe, registers MCP tools.
brain stop
Gracefully stop the Brain daemon.
brain status
Show current status: errors, solutions, modules, synapses, rules, insights.
brain query "error message"
Search for matching errors and solutions using full-text search + similarity matching.
brain modules [--project name]
List imported code modules. Filter by project to see what's been analyzed.
brain insights [--type trend]
View research insights: trends, gaps, synergies, templates, suggestions.
brain network [node-type] [id]
Explore the synapse network. Follow connections through spreading activation.
brain export [--format json]
Export Brain data for backup or analysis.
brain import <project-dir>
Import and analyze all source files from a project directory.
